Introduction

PathFinder visualizer is an interactive frontend web application that finds the path between 2 points- the start and the end point. The 2-D place act as a graph wherein each square (a node) on the grid can be either a start block, end block, wall block or simply a block through which travelling is possible.

PathFinder Visualizer not only finds the path but also calculates the shortes one between 2 nodes via different algorithms. The web application makes us visualise the process of finding the path through chosen algorithm and also calculates the time taken to do so. It also calculates the number of blocks that make the shortes path.

PathFinder Visualizer is also a game wherein one can create random grids and find path through algorithms.

Algorithms

The application currently gives 5 algorithm options to choose from. A quick information about each-

  • Depth First Search

    Depth-first search is a graph traversal algorithm that starts at the start node and explores as far as possible along each branch before backtracking.

    So the basic idea is to start from the start node and mark the node and move to the adjacent unmarked node and continue this loop until there is no unmarked adjacent node. Then backtrack and check for other unmarked nodes and traverse them. The traversal stops as soon as the end node is encountered. Finally print the nodes in the path.

    DFS does not necessarily yield shortest paths in graphs unless you follow the edge directly connecting the start and destination nodes.

    Time Complexity: O(V+E) , Space Compexity: O(V)

  • Breath First Search

    Breadth first search is a graph traversal algorithm that starts traversing the graph from root node and explores all the neighbouring nodes. Then, it selects the nearest node and explore all the unexplored nodes. The algorithm follows the same process for each of the nearest node until it finds the goal.

    BFS calculates the shortest paths in unweighted graphs only or when weights of each edge is same.

    Time Complexity: O(V+E) , Space Compexity: O(V)

  • Bi-Directional Breath First Search

    Bidirectional search is a graph search algorithm which find smallest path form source to goal vertex. It runs two simultaneous search â€“ Forward search form source/initial vertex toward goal vertex and Backward search form goal/target vertex toward source vertex

    The search terminates when two graphs intersect.

    Bidirectional search is fast and requires less memory but implementation is difficult.

    Time and Space Complexity: O(b^d) where b is branching factor and d is distance between start and end block

  • Dijkstra's algorithm

    It is a greedy algorithm that solves the single-source shortest path problem for both undirected and directed graph. The algorithm uses a greedy approach in the sense that we find the next best solution hoping that the end result is the best solution for the whole problem.

    In Djikstra, we overestimate the distance of each vertex from the starting vertex. Then we visit each node and its neighbors to find the shortest subpath to those neighbors.

    Time Complexity: O(E*log(V)) , Space Compexity: O(V)
